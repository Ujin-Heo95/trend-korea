## v0.8.0 (2026-03-29) — 스코어링 엔진 개편 (다중 팩터 랭킹)

### Added
- `engagement_snapshots` 테이블: 스크래퍼 매 실행 시 기존 게시글의 조회수/댓글수 이력 기록 (velocity 계산용)
- `source_engagement_stats` 테이블: 소스별 engagement Z-Score 정규화용 통계 캐싱
- 5가지 새 스코어링 팩터:
  - **Z-Score 정규화**: 소스별 상대 평가 (디시인사이드 vs 뉴스 스케일 불일치 해소)
  - **Velocity Bonus** [1.0–1.5]: engagement 증가 속도 기반 (2시간 윈도우 스냅샷 비교)
  - **Keyword Momentum** [1.0–1.3]: 3h/24h 키워드 빈도 가속도 (RSS 뉴스에도 적용)
  - **Trend Confirmation** [1.0–1.25]: 교차검증 시그널(convergence_score) 연동
  - **클러스터 보너스 개편** [1.0–2.5]: 로그 곡선 + 카테고리 다양성 (기존 최대 1.5 → 2.5)
- Zero-engagement Bayesian Prior: 고정 baseline 2.0 → 카테고리별 동적 중앙값 × credibility 가중치
- 스코어링 배치 실행 시 분포 로그 (p10/p50/p90/max)
- `cleanOldEngagementSnapshots()`: 6시간 초과 스냅샷 자동 정리

### Changed
- `computeScore()` 시그니처: 6개 개별 파라미터 → `ScoreFactors` 객체 패턴
- `calculateScores()`: 7개 서브 계산을 `Promise.all`로 병렬 실행 (각 실패 시 graceful fallback)
- `BaseScraper.saveToDb()`: INSERT 이후 `engagement_snapshots`에 배치 기록 추가

### Technical Notes
- `computeScoreLegacy()` 함수로 이전 공식 유지 (하위 호환)
- 스냅샷 실패 시 스크래핑 자체는 중단되지 않음 (try-catch 격리)
- 마이그레이션: `013_engagement_snapshots.sql` (2개 테이블)

## v0.6.1 (2026-03-29) — Gemini 쿼터 보호

### Fixed
- 키워드 추출 배치 크기 50→15, 동시성 2→1로 축소 (Gemini 무료 티어 15 RPM 초과 방지)
- 배치 간 딜레이 2초 추가 (분당 ~7 요청으로 안정화)
- 빈 결과 연속 시 early-exit 로직 추가 (쿼터 초과 감지 → 불필요한 요청 중단)
- `MAX_POSTS_PER_RUN` 500→200으로 축소

### Root Cause
- Gemini API 429 (쿼터 초과)로 `extractKeywords()`가 빈 배열만 반환
- `keyword_extractions` 테이블에 데이터 미적재 → `keyword_stats` 비어있음 → 핫이슈 태그 UI 미표시
