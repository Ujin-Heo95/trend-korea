# Changelog

## v0.6.1 (2026-03-29) — Gemini 쿼터 보호

### Fixed
- 키워드 추출 배치 크기 50→15, 동시성 2→1로 축소 (Gemini 무료 티어 15 RPM 초과 방지)
- 배치 간 딜레이 2초 추가 (분당 ~7 요청으로 안정화)
- 빈 결과 연속 시 early-exit 로직 추가 (쿼터 초과 감지 → 불필요한 요청 중단)
- `MAX_POSTS_PER_RUN` 500→200으로 축소

### Root Cause
- Gemini API 429 (쿼터 초과)로 `extractKeywords()`가 빈 배열만 반환
- `keyword_extractions` 테이블에 데이터 미적재 → `keyword_stats` 비어있음 → 핫이슈 태그 UI 미표시

## v0.6.0 (2026-03-29) — 교차 검증 트렌드 인텔리전스

### Added
- `trendCrossValidator.ts` — Google Trends × Naver DataLab × 커뮤니티 3중 교차 검증 서비스
- `trend_signals` 테이블 (마이그레이션 011) — 키워드별 convergence_score, signal_type 저장
- `GET /api/trends/signals` 엔드포인트 (type 필터, LRU 캐시 60초)
- `TrendRadar.tsx` 프론트엔드 컴포넌트 — 금색(확인됨)/파란색(구글만) 카드 UI
- `useTrendSignals()` React Query 훅 (60초 자동 갱신)
- Google Trends RSS 스크래핑 시 `metadata: { keyword, traffic, trafficNum }` 저장
- `cleanExpiredTrendSignals()` 만료 시그널 자동 정리

### Changed
- `scheduler/index.ts` — 20분 주기 교차 검증 cron 추가
- `cleanup.ts` — 정기 정리에 trend_signals 만료 행 삭제 포함
- `HomePage.tsx` — TrendRadar를 TrendingSection 위에 배치

### Architecture
- Convergence Score = log1p(googleTraffic) × naverMultiplier × communityMultiplier
- signal_type: 'confirmed' (구글+네이버 양성) / 'google_only' (구글만)
- Naver API 배칭: 5개/그룹, 500ms 딜레이, 일일 ~430회 (한도 25,000)
