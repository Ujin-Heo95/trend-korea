### 1.5 try-catch return [] 패턴 (P1)

**문제**: 일부 스크래퍼의 `fetch()` 내부에서 `catch { return [] }` → BaseScraper.run()의 retry + error 기록이 작동하지 않음. success_rate=100%인데 avg_posts=0인 "좀비" 소스 발생.

**사례**: KRX(400 LOGOUT), Google Trends(HTML 반환) — 둘 다 이 패턴으로 발견 지연.

**수정**: fetch() 내부 catch에서 throw해야 함.

**파일**: `backend/src/scrapers/krx.ts`, `backend/src/scrapers/google-trends.ts` (disabled이지만 패턴 참조)
