### 의사결정 기록 포맷

```markdown
# XX-NNN: 제목

- 상태: 승인 | 검토중 | 폐기
- 일자: YYYY-MM-DD
- 부서: 경영기획 | 개발 | 마케팅 | 법무

## 맥락
## 결정
## 대안
## 결과
```

## 문서 작성 원칙

1. **디렉토리명:** 영어 (`dev/`, `legal/`)
2. **파일명/내용:** 한글 (`아키텍처.md`)
3. **각 부서 README.md:** 해당 부서의 현황과 운영 가이드 역할
4. **하위 팀 폴더:** 실제 인원이 배치될 때 생성 (예: `dev/frontend/`)
