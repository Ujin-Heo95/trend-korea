## 1. SEO 전략

### 1.1 SPA SEO 한계와 해결

| 문제 | 영향 | 해결 (P1-01) |
|------|------|-------------|
| 순수 CSR (React SPA) | 네이버 Yeti, 카카오 스크래퍼가 JS 미실행 → 빈 HTML | 봇 UA 감지 프리렌더 미들웨어 |
| 모든 페이지 동일 meta | SNS 공유 시 제목/설명/이미지 동일 | 프리렌더에서 동적 meta 주입 |
| 정적 canonical (`/`) | 모든 페이지가 홈의 중복으로 인식 | 페이지별 동적 canonical |
| 정적 sitemap | /issue/:id, /daily-report/:date 미포함 | 백엔드 동적 sitemap 엔드포인트 (P1-02) |
| JSON-LD 없음 | 리치 스니펫/Google Discover 부적격 | Article, WebSite, BreadcrumbList (P1-03) |

**프리렌더 미들웨어 동작**:
1. 요청 UA가 봇 (`Googlebot`, `Yeti`, `kakaotalk-scrap`, `Twitterbot`, `facebookexternalhit`) → 서버에서 해당 페이지 데이터 fetch → HTML 템플릿에 동적 title/description/OG/canonical/JSON-LD 삽입 → 반환
2. 일반 사용자 → 기존 SPA 그대로 서빙
3. P4에서 Next.js SSR로 근본 교체 예정

### 1.2 검색엔진 등록 (P0-08)

| 플랫폼 | 한국 점유율 | 등록 방법 | 우선순위 |
|--------|-----------|-----------|---------|
| 네이버 서치어드바이저 | ~55% | DNS TXT 또는 meta 태그 | 최우선 |
| Google Search Console | ~35% | DNS TXT 또는 HTML 파일 | 최우선 |
| Daum 검색등록 | ~5% | 웹마스터 도구 | 선택 |

### 1.3 OG 이미지 동적 생성 (P1-04)

SNS 공유 시 클릭률을 결정하는 핵심 요소. 현재 OG 이미지 없음.

**접근**: satori + sharp → `/api/og-image/:postId`
- 이슈 상세: 제목 + 소스 배지 + 조회수 + 트렌드 스코어 on 브랜드 템플릿
- 일일 리포트: "오늘의 한국 인터넷 요약" + 날짜 + Top 5 키워드
- 홈: 브랜드 기본 이미지

### 1.4 Core Web Vitals (P2-10)

| 지표 | 현재 이슈 | 해결 |
|------|-----------|------|
| LCP | 외부 이미지 원본 크기 로드 | 이미지 프록시 + srcset (P2-11) |
| CLS | TrendHero 비동기 로드로 콘텐츠 밀림, 폰트 FOUT | min-height 예약, Inter 자체호스팅 |
| INP | 카테고리 전환 시 메인스레드 블로킹 | useTransition, content-visibility |

## 2. 사용자 획득

### 2.1 포지셔닝

**"네이버 실검 폐지 이후의 대안 — 한국 인터넷 실시간 온도계"**

- 네이버/다음: 뉴스만 → 우리: 뉴스 + 커뮤니티 + SNS + 정부 교차 분석
- 핵심 가치: "지금 한국에서 무엇이 뜨고 있는지" 30초 내 파악
- 차별점: 동일 주제 다중 소스 동시 감지 (교차 검증)

### 2.2 타겟 페르소나

| 페르소나 | 설명 | 핵심 니즈 |
|----------|------|-----------|
| 정보 중독자 | 다수 커뮤니티+뉴스 매일 확인 | 한 곳에서 모아보기 |
| 마케터/PR | 실시간 이슈 모니터링 | 크로스소스 트렌드 감지, 키워드 알림 |
| 해외 거주 한국인 | 한국 소식 놓치고 싶지 않음 | 일일 다이제스트 |
| 개발자/테크 관심자 | GeekNews + 커뮤니티 통합 | 테크 트렌드 큐레이션 |

### 2.3 런칭 3단계 (P1-08 ~ P1-10)

**1단계 (1-2주, 목표 DAU 20): 씨앗 사용자**

| 채널 | 행동 | 예상 유입 |
|------|------|-----------|
| Disquiet | 프로젝트 등록 + 런칭 포스트 (스크린샷 5장 + "네이버 실검 대안") | 100-300 방문 |
| GeekNews | "한국판 Trend Aggregator" 기술 아키텍처 글 | 50-200 방문 |
| 개인 SNS | X/Twitter, 페이스북 | 10-30 |

**2단계 (3-4주, 목표 DAU 50): 커뮤니티 침투**

| 채널 | 행동 | 예상 유입 |
|------|------|-----------|
| 네이버 블로그 | SEO 최적화 소개글 3편 (장문, 1000자+) | 검색 유입 누적 |
| DC 개발갤 | "사이드프로젝트 완성기" 기술 과정 공유 | 30-100 |
| 클리앙 모두의공원 | "사이드프로젝트 공유" (클리앙이 수집 소스라 관심↑) | 20-50 |

**3단계 (5-6주, 목표 DAU 100): 일일 바이럴 루프**
- X/Twitter 일일 트렌드 자동 봇 (매일 08:00 KST)
- "오늘 한국 인터넷에서 가장 핫한 키워드 5개" OG 이미지 카드 + 링크
- 카카오톡 채널 일일 다이제스트

**핵심 원칙**: weeklit.net 도메인 확보 완료. 외부 마케팅 즉시 착수 가능.

## 3. 리텐션 전략

### 3.1 현재 문제

- 사용자 → PostCard 클릭 → 외부 이탈 → 재방문 동기 없음 ("출구만 있는 건물")

## 4. 바이럴 메커니즘

### 4.1 바이럴 잠재력

| 콘텐츠 | 잠재력 | 현재 상태 | 필요 작업 |
|--------|--------|-----------|-----------|
| 일일 리포트 ("오늘의 한국 인터넷 요약") | **최고** | 공유 버튼 있음 | OG 이미지 생성 |
| 교차 검증 트렌드 (3곳 동시 급등) | 높음 | TrendRadar UI | 개별 공유 링크 |
| "N개 소스에서 화제" 클러스터 | 높음 | 이슈 상세 페이지 | 공유 최적화 |
| 영화/공연 랭킹 | 중간 | 포스터+외부링크 | 공유 버튼 |

현재 바이럴 계수(k-factor): **~0.05** (공유 버튼 있으나 OG 이미지 없어 CTR 낮음).
목표: OG 이미지 + 프리렌더 적용 후 **0.3+**.
